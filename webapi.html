1. What is .NET Core?

.NET Core is a cross-platform, open-source framework developed by Microsoft. It is used for building modern, cloud-based, and internet-connected applications. It supports Windows, macOS, and Linux platforms.

2. What are the main differences between .NET Core and .NET Framework?

.NET Core is cross-platform and open-source, whereas .NET Framework is limited to Windows and is closed-source. .NET Core is modular and has a smaller footprint, while .NET Framework includes a larger set of libraries and components. .NET Core also provides better performance and flexibility in deployment.

3. What is ASP.NET Core?

ASP.NET Core is a framework for building modern, high-performance, and cross-platform web applications. It is a part of .NET Core and includes features for web APIs, MVC, and Razor pages.

4. What is a Middleware in ASP.NET Core?

Middleware is a software component that is assembled into an application pipeline to handle requests and responses. Each component in the pipeline can perform operations before and after the next component in the pipeline.

5. What is Dependency Injection in ASP.NET Core?

Dependency Injection (DI) is a design pattern used to achieve Inversion of Control (IoC) between classes and their dependencies. ASP.NET Core has built-in support for DI, allowing you to inject services into your classes through constructors or methods.

6. How does routing work in ASP.NET Core?

Routing in ASP.NET Core maps incoming requests to the appropriate controller actions or endpoints. It is configured using route templates in the Startup class, and supports attribute routing and conventional routing.

7. What is Entity Framework Core?

Entity Framework Core (EF Core) is an object-relational mapper (ORM) that provides a high-level abstraction over database interactions. It allows you to work with databases using .NET objects and LINQ queries.

8. How do you handle configuration in .NET Core?

Configuration in .NET Core is handled using the Configuration API, which supports various sources such as JSON files, environment variables, command-line arguments, and user secrets. Configuration settings are accessed via the IConfiguration interface.

9. What is JWT?

JSON Web Token (JWT) is an open standard (RFC 7519) for securely transmitting information between parties as a JSON object. It is commonly used for authentication and authorization purposes.

10. What is the purpose of JWT in Web API?

JWT is used in Web API to authenticate users and authorize their access to resources. It allows stateless authentication, where the token contains the user's identity and claims, eliminating the need for server-side session storage.

11. How do you configure JWT authentication in ASP.NET Core?

JWT authentication in ASP.NET Core is configured in the Startup class using the AddAuthentication and AddJwtBearer methods. You need to specify the token validation parameters, including the issuer, audience, and signing key.

12. What are the key components of a JWT?

A JWT consists of three parts: the header, the payload, and the signature. The header contains metadata about the token, the payload contains claims or data, and the signature ensures the integrity of the token and verifies its authenticity.

13. What is the difference between Authentication and Authorization?

Authentication is the process of verifying the identity of a user, while Authorization determines what actions or resources the authenticated user is permitted to access. Authentication occurs before authorization in the security process.

14. How do you generate a JWT token in ASP.NET Core?

JWT tokens are generated using the JwtSecurityToken class and the JwtSecurityTokenHandler. You create a token with the desired claims and expiration, then serialize it to a string and return it to the client.

15. What are Claims in a JWT?

Claims are pieces of information encoded in a JWT payload that represent user attributes or roles. They can include standard claims (e.g., sub for subject) or custom claims specific to the application.

16. How do you validate a JWT token?

JWT tokens are validated using the JwtBearer middleware, which checks the token's signature, issuer, audience, and expiration date. The token's claims are also validated based on configured validation parameters.

17. What is ASP.NET Core Identity?

ASP.NET Core Identity is a membership system that provides functionality for user registration, authentication, and authorization. It includes features for managing users, roles, and claims, and integrates with ASP.NET Core's authentication system.

18. What is the role of the Startup class in ASP.NET Core?

The Startup class is used to configure the application's services and middleware. It contains the ConfigureServices method for setting up services and the Configure method for configuring the application's request pipeline.

19. What are Middleware Components in ASP.NET Core?

Middleware components are components in the request pipeline that handle HTTP requests and responses. They are added to the pipeline in a specific order and can perform tasks such as authentication, logging, and exception handling.

20. What is the purpose of the `ConfigureServices` method?

The `ConfigureServices` method in the Startup class is used to register services with the Dependency Injection (DI) container. It is where you configure and add services that are needed by the application, such as database contexts, authentication, and MVC services.

21. What is `Configure` method in ASP.NET Core?

The `Configure` method in the Startup class is used to define the application's request pipeline. It specifies how requests are handled and which middleware components are used in processing the requests.

22. What are Razor Pages in ASP.NET Core?

Razor Pages are a feature of ASP.NET Core that allows you to build dynamic web pages with a page-centric programming model. Each page is represented by a .cshtml file, which contains both the HTML markup and Razor code.

23. How do you handle exception handling in ASP.NET Core?

24. What is the purpose of `app.UseStaticFiles()` method?

25. What is `app.UseRouting()` in ASP.NET Core?

26. What is the difference between `IActionResult` and `ActionResult`?

`IActionResult` is an interface representing a result of an action method, while `ActionResult` is a concrete class that implements `IActionResult`. `ActionResult` provides additional convenience methods for returning different types of responses.

27. What is the purpose of `app.UseEndpoints()`?
