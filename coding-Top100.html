1. What is the difference between `ref` and `out` parameters in C#?

Answer:

The `ref` and `out` keywords are used to pass arguments by reference. However, there are some differences:

  • Initialization: Variables passed as `ref` must be initialized before being passed to the method, while variables passed as `out` do not need to be initialized.
  • Assignment: `ref` parameters need to be assigned a value before they are passed, while `out` parameters must be assigned a value within the method.
2. What is the difference between `IEnumerable` and `ICollection`?

Answer:

IEnumerable: Represents a forward-only cursor to iterate through a collection of items.

ICollection: Represents a collection of objects that can be individually accessed by index. It also provides additional methods for adding, removing, and checking for items.

3. How does garbage collection work in C#?

Answer:

C# uses an automatic garbage collection mechanism to manage memory. The garbage collector (GC) automatically releases memory that is no longer in use, helping to prevent memory leaks. It works by periodically identifying and freeing objects that are no longer referenced by the application.

4. What is the difference between `abstract` class and `interface` in C#?

Answer:

An `abstract` class can provide some implementation, whereas an `interface` cannot provide implementation. A class can inherit from only one `abstract` class but can implement multiple interfaces.

5. What is the purpose of the `using` statement in C#?

6. What is the difference between `String` and `StringBuilder` in C#?

Answer:

7. What are `delegates` in C#?

Answer:

8. How does exception handling work in C#?

Answer:

Exception handling in C# is done using `try`, `catch`, and `finally` blocks. The `try` block contains the code that might throw an exception, the `catch` block handles the exception, and the `finally` block is used for cleanup code that runs regardless of whether an exception was thrown.
