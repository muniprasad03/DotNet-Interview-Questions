Top 100 JavaScript and jQuery Interview Questions and Answers

1. What is JavaScript?

JavaScript is a high-level, dynamic, untyped, and interpreted programming language. It is commonly used for adding interactive features to web pages, making it essential for modern web development.

2. What is the difference between `var`, `let`, and `const`?

In JavaScript:

  • var: Declares a variable that is function-scoped or globally-scoped, and can be re-assigned.
  • let: Declares a block-scoped variable that can be re-assigned.
  • const: Declares a block-scoped variable that cannot be re-assigned once initialized.

3. What are JavaScript data types?

JavaScript has two types of data types:

  • Primitive types: undefined, null, boolean, number, string, symbol (introduced in ES6), and bigint (introduced in ES11).
  • Reference types: Objects, Arrays, Functions, Dates, etc.

4. What is a closure in JavaScript?

A closure is a function that retains access to its lexical scope, even when the function is executed outside that scope. Closures allow functions to have "private" variables.

Example:

function createCounter() {
    let count = 0;
    return function() {
        count++;
        return count;
    };
}
const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2

5. What is the difference between `==` and `===`?

== is the equality operator that performs type coercion before comparing values. === is the strict equality operator that compares both value and type without type coercion.

Example:

console.log(5 == '5');  // true
console.log(5 === '5'); // false

6. What is the purpose of `this` keyword in JavaScript?

The this keyword refers to the object that is currently executing the code. Its value is determined by the context in which a function is called.

Example:

const person = {
    name: 'John',
    greet: function() {
        console.log('Hello, ' + this.name);
    }
};
person.greet(); // Hello, John

7. What is event delegation in JavaScript?

Event delegation is a technique where you use a single event listener on a parent element to manage events for multiple child elements. It leverages event bubbling to catch events on elements that are dynamically added or removed.

Example:

document.getElementById('parent').addEventListener('click', function(event) {
    if (event.target && event.target.matches('button.child')) {
        console.log('Child button clicked');
    }
});

8. What is `async` and `await` in JavaScript?

async is a keyword that marks a function as asynchronous, allowing it to use await inside. await pauses the execution of the function until a Promise is resolved or rejected.

Example:

async function fetchData() {
    let response = await fetch('https://api.example.com/data');
    let data = await response.json();
    console.log(data);
}
fetchData();

9. What is a Promise in JavaScript?

A Promise is an object representing the eventual completion or failure of an asynchronous operation. It has three states: pending, fulfilled, and rejected.

Example:

let promise = new Promise((resolve, reject) => {
    setTimeout(() => resolve('Done'), 1000);
});
promise.then(result => console.log(result)); // Done

10. What is the purpose of `bind` method in JavaScript?

The bind method creates a new function that, when called, has its this keyword set to the provided value, with a given sequence of arguments preceding any provided when the new function is called.

Example:

11. What is the `call` method in JavaScript?

The call method calls a function with a given this value and arguments provided individually.

12. What is the `apply` method in JavaScript?

13. What is the difference between `null` and `undefined`?

null is an assignment value that represents no value or object. undefined means a variable has been declared but has not yet been assigned a value.

Example:

let a = null;
let b;
console.log(a); // null
console.log(b); // undefined

14. What is the difference between `forEach` and `map`?

forEach executes a provided function once for each array element, but does not return a new array. map creates a new array with the results of calling a provided function on every element in the calling array.

Example:

let numbers = [1, 2, 3];
numbers.forEach(n => console.log(n)); // 1 2 3
let doubled = numbers.map(n => n * 2);
console.log(doubled); // [2, 4, 6]

15. What is `event bubbling`?

Event bubbling is a type of event propagation in the DOM where the event starts at the target element and bubbles up to the root of the DOM tree. It allows event handlers to be attached to parent elements to handle events for child elements.

16. What is `event capturing`?

Event capturing (or trickling) is the opposite of event bubbling. The event starts from the root and propagates down to the target element. It can be used by specifying the third parameter as true in addEventListener.

Example:

document.getElementById('parent').addEventListener('click', function() {
    console.log('Captured!');
}, true);

17. What is `debouncing` in JavaScript?

Debouncing is a technique used to ensure that a function is only executed after a certain period of inactivity. It is commonly used to limit the rate at which a function is executed, such as in the case of handling user input events.

Example:

function debounce(func, delay) {
    let timer;
    return function(...args) {
        clearTimeout(timer);
        timer = setTimeout(() => func.apply(this, args), delay);
    };
}

18. What is `throttling` in JavaScript?

Throttling is a technique used to limit the rate at which a function is executed by ensuring that the function is called at most once within a specified time interval.

Example:

function throttle(func, limit) {
    let lastFunc;
    let lastRan;
    return function(...args) {
        const context = this;
        if (!lastRan) {
            func.apply(context, args);
            lastRan = Date.now();
        } else {
            clearTimeout(lastFunc);
            lastFunc = setTimeout(function() {
                if ((Date.now() - lastRan) >= limit) {
                    func.apply(context, args);
                    lastRan = Date.now();
                }
            }, limit - (Date.now() - lastRan));
        }
    };
}

19. What is `hoisting` in JavaScript?

Hoisting is a JavaScript behavior where variable and function declarations are moved to the top of their containing scope during the compile phase. This means you can use variables and functions before they are declared.

Example:

console.log(x); // undefined
var x = 5;

20. What is `prototype` in JavaScript?

Every JavaScript object has a prototype. A prototype is an object that is associated with every function and object by default. It is used to add properties and methods to objects and functions.

Example:

}

    console.log('Hello, ' + this.name);
};
const person = new Person('Alice');
person.sayHello(); // Hello, Alice

21. What is `JavaScript event loop`?

The JavaScript event loop is a mechanism that allows JavaScript to perform non-blocking operations by using an event-driven, single-threaded model. It continuously checks the call stack and the task queue, executing tasks from the queue when the call stack is empty.

22. What is `this` binding in JavaScript?

The value of this in JavaScript depends on how a function is called. It can be explicitly set using methods like bind, call, and apply, or it can be implicitly set based on the execution context.

23. What is `setTimeout` and `setInterval`?

setTimeout schedules a function to be executed after a specified delay, while setInterval repeatedly calls a function at specified intervals.

Example:

setInterval(() => console.log('Executed every 2 seconds'), 2000);

24. What is the `fetch` API?

The fetch API is a modern interface for making network requests. It provides a more powerful and flexible feature set compared to the older XMLHttpRequest.

25. What are `Promises` in JavaScript?

Promises are objects representing the eventual completion or failure of an asynchronous operation. They have three states: pending, fulfilled, and rejected. They provide a way to handle asynchronous operations in a more readable and maintainable way.

Example:

26. What is `strict mode` in JavaScript?

Strict mode is a way to opt into a restricted variant of JavaScript. It helps in catching common coding mistakes and unsafe actions, such as defining global variables unintentionally.

Example:

27. What is `destructuring` in JavaScript?

Destructuring is a convenient way of extracting values from arrays or properties from objects into distinct variables.

Example:

28. What are JavaScript modules?

JavaScript modules allow you to break up your code into separate files and manage dependencies between them. They can be imported and exported using import and export statements.

Example:
