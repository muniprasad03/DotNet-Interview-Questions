1. What is C#?

C# is a modern, object-oriented programming language developed by Microsoft as part of the .NET framework. It is used for developing a wide range of applications from web services to desktop applications.

2. What is the difference between a class and an object in C#?

A class is a blueprint or template for creating objects. It defines the data and behavior that the objects created from the class will have. An object is an instance of a class, created based on the class definition.

3. Explain the concept of inheritance in C#.

Inheritance is a fundamental concept in object-oriented programming where a new class (derived class) is based on an existing class (base class). The derived class inherits the fields and methods from the base class and can also have additional fields and methods of its own.

4. What is the difference between "ref" and "out" parameters in C#?

The "ref" keyword is used to pass a parameter by reference, allowing the method to modify the value of the parameter. The "out" keyword is similar but is used to indicate that the parameter will be assigned a value within the method, and it must be assigned before the method exits.

5. What is the purpose of the "using" statement in C#?

The "using" statement is used to ensure that resources are properly disposed of after they are no longer needed. It is typically used with objects that implement the IDisposable interface, such as file streams or database connections.

6. What are delegates in C#?

Delegates are objects that represent references to methods. They are used to pass methods as arguments to other methods and to define callback methods. Delegates are type-safe and secure function pointers.

7. What is a lambda expression in C#?

A lambda expression is a concise way to represent an anonymous method. It is used primarily in LINQ queries and allows for the creation of inline delegate instances. Lambda expressions use the syntax `()=>{}` and can capture variables from their enclosing context.

8. What is the difference between an interface and an abstract class in C#?

An interface defines a contract that classes must implement, but it does not provide any implementation details. An abstract class, on the other hand, can provide some method implementations and can contain fields. Abstract classes are used when classes share common functionality, while interfaces are used to define a contract that multiple classes can implement.

9. What are generics in C#?

Generics allow you to define classes, interfaces, and methods with a placeholder for the type they operate on. This enables you to create type-safe data structures and methods without sacrificing performance or type safety. Generics are used to create collections and other types that work with any data type.

10. What is the difference between "async" and "await" in C#?

The "async" keyword is used to define a method as asynchronous, allowing it to perform tasks without blocking the main thread. The "await" keyword is used within an "async" method to pause the execution until a task is completed, enabling asynchronous operations to be written in a synchronous-like manner.

11. What is a "Task" in C# and how does it differ from a "Thread"?

A "Task" represents an asynchronous operation that can be awaited and is part of the Task Parallel Library (TPL). It provides a higher-level abstraction for managing asynchronous operations compared to raw threads. A "Thread" represents a lower-level unit of execution and requires more manual management compared to Tasks.

12. How does dependency injection work in C#?

Dependency Injection (DI) is a design pattern used to achieve Inversion of Control (IoC) by injecting dependencies into a class rather than the class creating its own dependencies. In C#, DI is commonly used with frameworks like ASP.NET Core to manage service lifetimes and facilitate testing and maintainability.

Value types hold data directly and are typically stored on the stack (e.g., int, float, struct). Reference types hold a reference to the data, which is stored on the heap (e.g., class, string, array).
