1. What are design patterns and can you name a few commonly used ones?

Design patterns are general repeatable solutions to common problems in software design. They are not ready-made solutions but templates for how to solve problems in various situations.

Some commonly used design patterns include:

  • Singleton: Ensures a class has only one instance and provides a global point of access to it.
  • Factory: Creates objects without specifying the exact class of object that will be created.
  • Repository: Mediates between the domain and data mapping layers using a collection-like interface for accessing domain objects.
  • Decorator: Adds behavior to objects dynamically without altering their structure.
  • Observer: Defines a one-to-many dependency between objects so that when one object changes state, all its dependents are notified and updated automatically.

3. What is a design pattern in C# and can you describe the Singleton pattern?

A design pattern is a general reusable solution to a commonly occurring problem within a given context in software design. The Singleton pattern is a type of design pattern that ensures a class has only one instance and provides a global point of access to it.

Implementation of Singleton Pattern:

public class Singleton
{
private static Singleton instance;
private static readonly object lockObject = new object();

private Singleton() { }

public static Singleton Instance
{
    get
    {
        if (instance == null)
        {
            lock (lockObject)
            {
                if (instance == null)
                {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
}

4. How does garbage collection work in .NET?

Garbage collection in .NET is an automatic memory management feature that helps reclaim memory occupied by objects that are no longer in use. The .NET garbage collector (GC) works in the following way:

  • Generational Approach: Objects are divided into three generations based on their lifetime (Gen 0, Gen 1, Gen 2). New objects start in Gen 0. If they survive a GC cycle, they move to Gen 1 and then Gen 2.
  • Mark and Sweep: During garbage collection, the GC marks all live objects and then sweeps away the unmarked objects.
  • Compaction: The GC may compact memory to reduce fragmentation and make allocation more efficient.
